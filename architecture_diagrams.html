9. Frontend: which component is mounted, and why

There's no real "mounting" (no React/Vue) — quiz.html renders all three cards into the DOM at once, and quiz.js just toggles Bootstrap's d-none class to show/hide them. Only one is visible at a time.

Element IDVisible whenWho toggles it
#quizCardDefault state — always visible until quiz finishesNever hidden until showResults() runs
#resultsCardAfter the last question is answered and /next returns finished:trueshowResults() removes its d-none
#questionnaireCardSame moment as resultsCard — shown togethershowResults() removes its d-none too
#feedbackBoxAfter an answer is submitted, inside quizCardsubmitAnswer()'s .then() callback
#nextBtnSame moment as feedbackBoxsubmitAnswer()'s .then() callback
#thankYouBoxAfter questionnaire POST succeedsquestionnaire form's submit handler
Practical debugging tip: if something looks "stuck" (e.g. "Loading question..." never goes away), open DevTools Console (F12). The whole quiz.js file is one IIFE — if any line throws (e.g. document.getElementById(...) returns null because an element was renamed/removed in quiz.html but quiz.js still references the old ID), the script stops executing entirely, and the final renderQuestion(currentQuestion) call at the bottom never runs. That's why quiz.html and quiz.js must always be replaced together, never one without the other.
Global state lives in plain JS variables at the top of the IIFE (currentQuestion, questionNumber, totalQuestions, difficulty, answered) — there's no framework state management. Every AJAX response reassigns these variables directly before calling renderQuestion() again.
